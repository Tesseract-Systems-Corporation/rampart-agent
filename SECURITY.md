# Security Policy

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability in the Rampart Agent, please report it responsibly.

**Do not open a public GitHub issue** to report security concerns.

Instead, report vulnerabilities by emailing:

**security@gotesseract.com**

Please include, where possible:
- A description of the vulnerability
- Steps to reproduce
- Affected versions or configurations
- Any relevant logs, screenshots, or proof-of-concept code

Reports will be acknowledged and reviewed by the Tesseract Systems Corporation engineering team.

---

## Disclosure Process

Upon receipt of a valid security report:

1. We will acknowledge the report within a reasonable timeframe.
2. The issue will be reviewed and triaged to assess impact and severity.
3. If confirmed, we will work to develop and test a fix.
4. Security updates will be released according to severity and operational impact.

We coordinate disclosure responsibly and do not publish vulnerability details prior to a fix being available.

---

## Supported Versions

Security updates are provided for the most recent released versions of the Rampart Agent.

Users are encouraged to stay current with released versions to ensure timely security fixes.

---

## Security Scope

This repository covers the **Rampart Agent** only.

The Rampart control plane and associated services are proprietary systems and are not part of this repository.

---

## Safe Harbor

Tesseract Systems Corporation supports good-faith security research conducted in accordance with responsible disclosure practices.

We will not pursue legal action against researchers who:
- Make a good-faith effort to avoid privacy violations or data destruction
- Do not exploit vulnerabilities beyond proof-of-concept
- Report findings responsibly as outlined above
